Features

Fast, safe, and honest about everything.

Built on the principles you'd want from a cleaner: no black box, no permanent deletes, no surprises.

Grouped by app

Library/Caches/com.apple.Safari/* collapses into a single "Safari" row — 10,000 files, one decision.

Moves to Trash

No permanent deletion. Everything goes to the Finder Trash so you can restore anything, anytime.

Dry run by default

The first time you run MacSift, cleaning is simulated. Nothing is touched until you explicitly opt in.

Full transparency

Every row shows the app, category, file count, and total size. Click any group to inspect the top files and reveal in Finder.

Liquid Glass UI

Built on the new macOS 26 Tahoe APIs. Native sidebar, inspector, materials, and toolbar — feels at home next to Finder.

Fast and responsive

Parallel async scans, smart progress throttling, and pre-sorted caches keep the UI snappy even with 50,000+ files.

Install

Two commands. No Xcode project required.

MacSift is a SwiftPM project that builds into a proper .app bundle with the provided script.

1

Clone the repo

git clone https://github.com/Lcharvol/MacSift.git
cd MacSift
2

Build & launch

./build-app.sh
open MacSift.app
Requires macOS 26 Tahoe, Xcode 26 toolchain, and Full Disk Access (granted at first launch).
FAQ

Questions you'd ask a friend.

Will MacSift delete anything important?

No — MacSift never permanently deletes. Everything goes to the Trash via FileManager.trashItem. Dry run is on by default; destructive actions require an explicit confirmation, and there's an extra warning above 10 GB. System paths (/System, /usr, /bin, /sbin) and still-installed-app data are hard-blocked.

Why does it need Full Disk Access?

To scan /private/var/log and some system caches. Without FDA, the app still works — it just skips the protected paths and shows a banner. You grant it once in System Settings.

Is there a DMG or Homebrew cask?

Not yet. MacSift is a personal project and not notarized for distribution, so you build it yourself from source with the provided script. The build is one command and produces a proper .app bundle.

Does it collect any telemetry?

No. Zero. It never makes a network connection. All scanning and cleaning happens entirely on your Mac.

How fast is it?

On a typical Mac with ~50,000 scannable files, a full scan completes in 3–8 seconds. The UI stays responsive throughout — scanning is parallelized across categories, progress is throttled, and sorting runs off the main thread.

Does it support older macOS versions?

No — MacSift uses the Liquid Glass APIs introduced in macOS 26 Tahoe. Supporting older versions would mean rewriting the UI layer, which isn't planned.
